    运维手册

    本页面介绍如何运维和管理本手册，包括本地使用、内容管理、配置修改和部署等操作。

    1. 本地使用

    安装依赖

    在本地使用前，需要安装必要的依赖：

    pip install mkdocs mkdocs-material mkdocs-minify-plugin

    本地预览

    修改内容后，可以通过以下命令启动本地服务器进行预览：

    mkdocs serve

    然后在浏览器中访问 http://localhost:8000 查看实时预览。

    构建站点

    在部署前，可以在本地构建站点，检查是否有错误：

    mkdocs build

    2. 新增文章

    创建文章文件

    1. 在 docs 目录下创建新的 Markdown 文件（例如 docs/new-article.md）
    2. 在文件中添加内容，使用 Markdown 语法编写

    添加导航

    在 mkdocs.yml 文件的 nav 部分添加对应的导航条目：

    nav:
  - 首页: index.md
  - 新增文章: new-article.md  # 添加新文章

    3. 新增栏目

    创建栏目目录

    1. 在 docs 目录下创建新的文件夹（例如 docs/new-section/）
    2. 在该文件夹中创建 index.md 文件作为栏目首页
    3. 可以在该文件夹中添加其他文章文件

    添加导航

    在 mkdocs.yml 文件的 nav 部分添加新的栏目：

    nav:
  - 首页: index.md
  - 新栏目: new-section/  # 添加新栏目
    - 栏目首页: new-section/index.md
    - 子页面1: new-section/page1.md
    - 子页面2: new-section/page2.md

    4. 新增图片

    创建图片文件夹

    在 docs 目录下创建 images 文件夹（如果不存在）：

    mkdir -p docs/images

    添加图片

    1. 将图片文件放入 docs/images 文件夹中
    2. 可以根据栏目或功能创建子文件夹，分类存储图片

    引用图片

    在 Markdown 文件中使用以下语法引用图片：

    ![图片描述](images/example.png)

    路径引用方式

    • 相对路径：从当前 Markdown 文件所在目录开始的路径 markdown ![图片描述](../images/example.png) # 上级目录的 images 文件夹

    • 绝对路径：从 docs 目录开始的路径（推荐） markdown ![图片描述](/images/example.png) # 从根目录开始的路径

    图片优化

    • 压缩图片：使用图片压缩工具（如 TinyPNG、Squoosh 等）减小图片文件大小
    • 选择合适的格式：根据图片类型选择合适的格式（如 JPG 用于照片，PNG 用于透明背景图片）
    • 控制图片尺寸：根据页面布局需要调整图片尺寸

    5. 修改各种的显示名字

    修改站点名称

    在 mkdocs.yml 文件中修改 site_name 字段：

    site_name: IE Arch Handbook  # 修改为你的站点名称

    修改站点 URL

    在 mkdocs.yml 文件中修改 site_url 字段：

    site_url: https://h4nkx.github.io/  # 修改为你的站点 URL

    修改栏目名称

    直接修改 mkdocs.yml 文件中 nav 部分的栏目名称：

    nav:
  - 首页: index.md
  - 原栏目名: section/  # 修改为新的栏目名

    修改文章标题

    直接修改 Markdown 文件中的标题：

    # 原标题  # 修改为新的标题

    6. 如何部署 GitHub

    自动部署

    1. 确保 .github/workflows/deploy.yml 文件已配置正确
    2. 将修改推送到 main 分支
    3. GitHub Actions 会自动执行构建和部署

    手动部署

    运行以下命令手动构建并推送更新：

    mkdocs gh-deploy

    部署验证

    部署完成后，可以通过以下 URL 访问站点：

    https://H4nkX.github.io/H4nkX.github.io/

    部署问题排查

    • 检查 GitHub Actions 工作流是否成功执行
    • 检查 gh-pages 分支是否存在且包含站点文件
    • 检查 GitHub Pages 设置是否正确（使用 gh-pages 分支）
