# RPA（机器人流程自动化）

> 用代码代替人类重复劳动，让机器人帮你干活

RPA（Robotic Process Automation）是通过软件机器人模拟人类在计算机上的操作，自动执行重复性、规则明确的业务流程。

---

## 一、Python RPA 必用库（企业 100% 用到）

### 1. 网页自动化（最核心、必须有）

| 库 | 用途 | 特点 |
|----|------|------|
| **Selenium** | 浏览器自动化 | 企业级标准、最稳定、所有系统都兼容 |
| **Playwright** | 现代浏览器自动化 | 新一代、自动等待、AI 自动化首选 |

## 二、AI 时代：你不用写代码了！

### 你现在的工作方式：

1. **你告诉 AI 需求**
   - "帮我写一个自动登录网页、下载报表的 RPA"

2. **AI 直接写代码**
   - 用上面所有库自动组合

3. **你直接复制运行**
   - 不用懂代码、不用记库、不用学语法

4. **AI 帮你改 bug**
   - 报错直接丢给 AI，AI 自动修复

### 一句话总结：

> **库 = 工具**  
> **AI = 帮你拿工具的人**  
> **你 = 指挥者**

## 三、AI 和 RPA 在这个时代的真正关系

| 场景 | Agent（AI 代理） | RPA |
|------|------------------|-----|
| **可审计、可回溯** | ❌ 做不到 | ✅ 完全可记录 |
| **无头服务器运行** | ❌ 需要界面 | ✅ 后台静默执行 |
| **批量处理 1000 次** | ❌ 几小时 | ✅ 几分钟 |
| **对接系统接口** | ❌ 能力有限 | ✅ 可以对接 |

### 核心结论

> **AI 不会取代 RPA，而是让 RPA 更强大。**
> 
> AI 负责"思考"，RPA 负责"执行"。
> 
> 企业需要稳定、可控、可审计的自动化——这就是 RPA 不可替代的价值。

## 五、快速开始

### 最小可行代码示例

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
import time

# 启动浏览器
driver = webdriver.Chrome()

# 打开网页
driver.get("https://example.com")

# 找到输入框并输入
driver.find_element(By.ID, "username").send_keys("your_username")
driver.find_element(By.ID, "password").send_keys("your_password")

# 点击登录
driver.find_element(By.ID, "login").click()

# 等待加载
time.sleep(3)

# 下载报表
driver.find_element(By.ID, "download").click()

print("RPA 任务完成！")
```
